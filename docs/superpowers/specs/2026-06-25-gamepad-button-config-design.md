# Настройка кнопок геймпада + кнопка «назад» на экране кликера

Дата: 2026-06-25

## Цель

Дать пользователю переназначать кнопки геймпада на три действия матча
(очко команде A, очко команде B, отмена последнего очка) прямо с экрана
кликера, а также добавить навигацию «назад» с этого экрана.

## Контекст

Сейчас маппинг кнопок захардкожен в `lib/gamepad/mapping.ts`
(`L1=4 → pointA`, `R1=5 → pointB`, `✕=0 → undo`). Действия матча уже
реализованы (`point(0)`, `point(1)`, `undoPoint()` в
`GamepadController.tsx`). Экран `/clicker` уже рисует три строки
назначения (`ModeRow`), но их `onClick` пустые. Кнопки «назад» на экране
нет.

Геймпад слушают **два** смонтированных компонента: `GamepadController`
(игровая логика) и экран `/clicker` через тот же поллинг. Координация
режима обучения идёт через флаг в сторе.

## Решения по требованиям

- **Назначаемые кнопки:** любая кнопка геймпада (по индексу Gamepad API).
- **Конфликт:** при назначении кнопки, уже привязанной к другому
  действию, она снимается со старого действия (дублей не бывает).
- **Сброс к дефолтам:** не нужен.
- **Выход из режима обучения:** только повторным тапом по той же строке
  (без авто-таймаута).

## Архитектура

### Хранилище — `store/clickerStore.ts`

- Новое персистимое поле:
  `bindings: Record<ClickerAction, number>` с дефолтом
  `{ pointA: 4, pointB: 5, undo: 0 }`. Добавляется в `partialize`.
- Новое рантайм-поле (НЕ персистится): `learning: ClickerAction | null`.
- Экшены:
  - `setBinding(action: ClickerAction, index: number)` — записывает
    привязку и снимает `index` с любого другого действия, у которого он
    был (разрешение конфликта).
  - `setLearning(action: ClickerAction | null)`.

Логика разрешения конфликта выносится в чистую функцию
`resolveBindings(current, action, index)` (тестируемую без стора).

### Маппинг — `lib/gamepad/mapping.ts`

- `buttonToAction(index, bindings)` — обратный поиск по объекту
  `bindings` (раньше была чистая функция от номера с константами).
- `EVENT_LABEL` сохраняется как подписи действий, но без привязки к
  конкретным кнопкам (текст вида «L1/R1» уходит в подпись кнопки).
- Новый хелпер `buttonLabel(index): string` — человекочитаемое имя
  кнопки по индексу: `0 → "✕ / A"`, `4 → "L1"`, `5 → "R1"`, иначе
  `"Кнопка N"`.

### Игровая логика — `components/GamepadController.tsx`

- Читает `bindings` и `learning` из стора.
- В `handleButtonDown(index)`:
  - если `learning !== null` → вызвать `setBinding(learning, index)` и
    `setLearning(null)`; очко НЕ засчитывается;
  - иначе → `buttonToAction(index, bindings)` и, как сейчас,
    `point/undo` + `setLastEvent`.

### Экран кликера — `app/clicker/page.tsx`

- Кнопка «назад» в шапке (`router.back()`), слева от заголовка.
- Три `ModeRow` становятся интерактивными:
  - тап по строке → `setLearning(action)`;
  - строка в режиме обучения показывает подсказку «нажмите кнопку…»;
  - повторный тап по той же строке → `setLearning(null)`;
  - подзаголовок строки показывает текущую привязанную кнопку через
    `buttonLabel(bindings[action])`.

## Поток данных

Тап по строке → `learning = pointA` → пользователь жмёт кнопку на
геймпаде → `GamepadController` ловит `onButtonDown`, видит `learning`,
вызывает `setBinding('pointA', idx)` (снимая `idx` с других действий) и
сбрасывает `learning`. UI реактивно обновляет подпись.

## Тестирование

- `lib/gamepad/__tests__/mapping.test.ts`: обновить под новую сигнатуру
  `buttonToAction(index, bindings)`.
- Добавить тесты на `resolveBindings`: назначение свободной кнопки;
  назначение кнопки, занятой другим действием (снятие со старого);
  переназначение того же действия на ту же кнопку.

## Вне области

- Сброс к дефолтам.
- Назначение нескольких кнопок на одно действие.
- Настройка осей/стиков, длинных нажатий, комбинаций.
