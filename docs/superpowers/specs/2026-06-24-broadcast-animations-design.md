# Анимации счёта, направление подачи и чистка входа

Дата: 2026-06-24

## Цель

Усилить визуал и убрать шероховатости UX:

1. Сделать счёт в трансляции крупнее и заметнее.
2. Анимировать смену счёта, победу в гейме и в сете (через `motion`,
   эффекты в духе react-bits).
3. Показать направление подачи пунктиром (диагональ в принимающий квадрат) —
   на корте `/match` и компактной схемой на `/broadcast`.
4. Убрать дефолтные имена игроков (пустые поля с плейсхолдерами).
5. Убрать бесполезный крестик `✕` на главной, заменить иконкой входа в
   геймпад (`/clicker`); добавить такую же иконку в шапку `/match`.

## Контекст и ограничения

- Стек: Next.js 16 (App Router), React 19, Zustand, Tailwind v4, Vitest.
- Анимаций-библиотеки нет; текущие эффекты — ручной CSS
  (`animate-pulse2`, `animate-ring`). Добавляем одну зависимость — `motion`
  (наследник framer-motion, поддерживает React 19). Компоненты с `motion`
  должны быть client-компонентами — `/broadcast`, `/match`, `ScoreBoard`
  уже `"use client"`.
- Очки в паделе — это метки (`0/15/30/40/Ad`), а не инкремент целых, поэтому
  анимация счёта — это keyed-смена метки, а не счётчик CountUp.
- Конфетти реализуем на `motion` (≈14 частиц), без отдельной библиотеки —
  чтобы не раздувать бандл PWA.
- `/clicker` сейчас недостижим: на него ничего не ведёт.

## Архитектура

Изоляция: «что произошло» отделено от «как это анимировать».

### `lib/padel/scoreEvents.ts` — чистое определение событий

- `diffMatch(prev: MatchState | null, curr: MatchState): ScoreEvent`
- `ScoreEvent = { pointChanged: boolean; gameWonBy?: TeamIndex;
  setWonBy?: TeamIndex; matchWon: boolean }`
- Логика на сравнении снимков:
  - сумма геймов в текущем сете выросла у команды → `gameWonBy`;
  - число сетов выросло у команды → `setWonBy`;
  - `status` стал `"completed"` → `matchWon`;
  - иначе при любом изменении очков/счёта → `pointChanged`.
- Чистая, покрывается юнит-тестом (TDD).

### `lib/padel/serve.ts` — цель подачи (расширение существующего)

- `serveTarget(server: CourtPos): { x: "left"|"right"; y: "top"|"bottom" }`
- Подача всегда идёт через сетку по диагонали в противоположный квадрат:
  left-top → right-bottom, left-bottom → right-top, right-top → left-bottom,
  right-bottom → left-top. Детерминируется позицией подающего. Юнит-тест.

### `lib/padel/useScoreEvents.ts` — хук-обёртка

- `useScoreEvents(match): ScoreEvent` — хранит предыдущий снимок в `useRef`,
  на каждый рендер вызывает `diffMatch(prevRef.current, match)` и обновляет
  ref. Не знает про рендер celebration — только отдаёт событие.

## Компоненты

### `components/AnimatedPoint.tsx`

- Оборачивает метку очка в `motion` + `AnimatePresence`, key = значение очка.
- Уходящая метка: slide/fade вверх; входящая: slide снизу + кратковременная
  вспышка scale+glow.
- Проп `size`/вариант: крупный для `/broadcast`, компактный для ячейки очка
  в `ScoreBoard`.

### `components/WinCelebration.tsx`

- Управляется `useScoreEvents`.
- `gameWonBy` → короткий glow-burst (pulse) на стороне победителя, ~1 с.
- `setWonBy` → крупный баннер («Сет · Команда A») + конфетти-всплеск на
  `motion` (≈14 частиц, разлёт наружу, fade). На `/broadcast` — богаче, на
  `/match` — легче.
- Автоскрытие по таймеру; уважает повторные события (сброс таймера).

### `components/ServeMiniCourt.tsx`

- Компактная SVG-схема подачи для нижней полосы `/broadcast`: тот же
  пунктир «подающий → принимающий квадрат», что и на корте матча.

### Иконка входа в геймпад

- Иконка `Gamepad2` из `lucide-react` (уже в зависимостях).
- На главной заменяет `✕` в шапке, по тапу → `/clicker`.
- В шапке `/match` — рядом с кнопкой «Трансляция», → `/clicker`.

## Изменения экранов

### `/broadcast`

- Очко крупнее: 132px → больше (responsive clamp, ориентир ~160–180px),
  усиленный glow у подающей стороны.
- Очко рендерится через `AnimatedPoint`.
- `WinCelebration` (богатый вариант) поверх.
- `ServeMiniCourt` в нижней полосе.

### `/match`

- `AnimatedPoint` в ячейке очка `ScoreBoard`.
- `WinCelebration` (лёгкий вариант).
- `CourtDiagram`: пунктирная диагональ подачи + подсветка целевого квадрата.
- Иконка геймпада в шапке.

### `CourtDiagram`

- Добавить dashed-линию accent от маркера подающего к целевому квадрату
  (`serveTarget`) и мягкую подсветку этого квадрата.

### Главная (`/`)

- Убрать `✕`; на его месте — иконка геймпада → `/clicker`.
- Дефолтные имена убрать: `a`/`b` стартуют как `["", ""]`, у инпутов
  плейсхолдеры «Игрок 1» / «Игрок 2». Пустые имена при старте матча
  заменяются на дженерик («Игрок 1/2»), чтобы счёт/трансляция не показывали
  голый «/». Аватар при пустом имени уже показывает «·».

### `/clicker`

- Фолбэк-строка «Алекс · Команда A» при отсутствии матча → дженерик
  («Команда A»), без выдуманного имени.

## Вне рамок (YAGNI)

- Звук на победу.
- Настраиваемые темы/цвета анимаций.
- Полноценная физика конфетти / отдельная библиотека частиц.
- Анимация перемещения игроков по квадратам при смене стороны deuce/ad
  (диаграмма остаётся с фиксированными квадрантами).
