# RALLY — приложение для счёта в паделе. Дизайн-спецификация

Дата: 2026-06-23
Источник дизайна: Claude Design проект «Приложение для счета в паделе» (`Padel Score.dc.html`), 4 экрана.

## 1. Цель и рамки

Установимое мобильное PWA для ведения счёта в матче по паделу. Работает полностью на клиенте,
офлайн. Ядро — чистый движок счёта (state machine), не зависящий от React. UI — тонкий слой поверх.

В этой итерации:

- Полностью рабочий движок счёта (очки → геймы → сеты → матч), настраиваемый форматом.
- 4 экрана из дизайна + минимальный оверлей завершения матча.
- Экран кликера — визуальный макет; настройки сохраняются в состоянии, но реальный Web Bluetooth
  **не реализуется** (заглушка на будущее).
- Автосохранение текущего матча в `localStorage` для восстановления после перезагрузки/закрытия.

Не входит: реальное BLE-подключение, аккаунты, облачная синхронизация, история матчей, бэкенд.

## 2. Технологический стек

- **Next.js (App Router)** + TypeScript.
- **Tailwind CSS v4** (CSS-first: токены в `@theme` внутри `globals.css`, без `tailwind.config.ts`).
- **shadcn/ui** (v4, стиль base-nova) как библиотека базовых компонентов (Switch, Input, Dialog,
  Button); фирменный лаймовый вид остаётся на наших токенах, shadcn даёт доступность и поведение.
  Кастомные компоненты (ScoreBoard, CourtDiagram, PhoneScreen, SegmentedControl) — собственные.
- **Zustand** для состояния приложения (обёртка над чистым движком), с persist в `localStorage`.
- **Serwist (@serwist/next)** — service worker, офлайн-кеш, установимость (Turbopack/Next 16).
  Манифест — через `app/manifest.ts`.
- Шрифты через `next/font`: **Manrope** (дисплей; есть кириллица) + **JetBrains Mono** (метки/цифры).
  Замена Archivo → Manrope: Archivo не содержит кириллических глифов, а интерфейс на русском.
- Платформа: **Next.js 16** (App Router, Turbopack), **React 19**.
- **Vitest** для юнит-тестов движка (разработка движка ведётся test-first / TDD).

## 3. Архитектура

```
app/
  layout.tsx          → шрифты, метаданные PWA, глобальные стили
  page.tsx            → 01 Новый матч (настройка)
  clicker/page.tsx    → 02 Кликер (макет)
  match/page.tsx      → 03 Активный матч
  broadcast/page.tsx  → 04 Трансляция (landscape)
lib/padel/            → чистый движок (без React, покрыт юнит-тестами)
  types.ts            → Config, Team, Player, MatchState, PointEvent, ...
  rules.ts            → константы и предикаты правил (очки, win-by-2, golden point, tiebreak)
  engine.ts          → createMatch, scorePoint, undo, reset (чистый редьюсер + история)
  serve.ts            → ротация подачи, сторона (deuce/ad), позиции игроков на корте
  format.ts           → отображаемые лейблы (0/15/30/40/AD, счёт сетов и т.п.)
store/
  matchStore.ts       → Zustand store: state + dispatch(scorePoint/undo/reset), persist
  clickerStore.ts     → настройки кликера (режим кнопок, владелец) — заглушка BLE
components/           → ScoreBoard, CourtDiagram, ScoreButtons, StatusBar, формат-тогглы, и т.д.
```

Принцип: движок — чистые функции `state → state`, его можно понять и протестировать изолированно.
Store только хранит состояние и вызывает функции движка. UI читает состояние из store.

## 4. Движок счёта (ядро)

Иммутабельный редьюсер над `MatchState` со стеком истории для отмены. Все правила управляются
конфигом из экрана настройки.

### Конфиг матча
- `sets`: best of — 1 / 3 / 5 (по умолчанию 3).
- `gamesPerSet`: первый до N геймов — 4 / 6 / 9 (по умолчанию 6), win by 2.
- `goldenPoint`: bool (по умолчанию вкл) — при 40-40 один решающий мяч.
- `tiebreak`: bool (по умолчанию вкл) — при N–N тай-брейк до 7 (win by 2).

### Правила
- **Очки в гейме:** 0 → 15 → 30 → 40 → гейм. При 40-40: если `goldenPoint` — следующий мяч
  выигрывает гейм; иначе преимущество (AD) и win-by-2.
- **Геймы в сете:** первый до `gamesPerSet`, win by 2. При `gamesPerSet`–`gamesPerSet`:
  если `tiebreak` — тай-брейк до 7 очков (win by 2); иначе продолжается win-by-2 по геймам.
- **Сеты в матче:** матч заканчивается, когда команда выигрывает большинство из `sets`
  (1 → 1 сет, 3 → 2 сета, 5 → 3 сета).
- **Подача и корт:** подача чередуется между командами каждый гейм и проходит по фиксированному
  порядку всех 4 игроков. Сторона подачи: правый квадрат (deuce) при чётной сумме очков, левый (ad)
  при нечётной. `serve.ts` выводит подающего, сторону и позиции для диаграммы корта.
- **Отмена (undo):** каждый разыгранный мяч кладёт снимок состояния в историю; undo снимает
  последний. Полностью обратима, включая переходы через границы гейма/сета.
- **Завершение матча:** при победе фиксируется победитель; UI показывает оверлей завершения.

### Состояние (эскиз)
```ts
type PointLabel = '0' | '15' | '30' | '40' | 'AD';
interface TeamScore { points: number; games: number[]; sets: number; } // games[setIndex]
interface MatchState {
  config: Config;
  teams: [Team, Team];
  score: [TeamScore, TeamScore];
  currentSet: number;
  serving: { team: 0 | 1; player: 0 | 1; side: 'deuce' | 'ad' };
  inTiebreak: boolean;
  status: 'in_progress' | 'completed';
  winner?: 0 | 1;
  history: MatchState[]; // снимки для undo (без вложенной истории)
  startedAt: number;     // для таймера матча
}
```

### Тесты (test-first, Vitest)
Покрываем до написания UI: прогресс очков; deuce и golden point; тай-брейк (вход, набор до 7,
win-by-2); завершение гейма/сета/матча; ротация подачи и сторона; undo через все границы.

## 5. Экраны и навигация

Раздельные роуты Next.js. Поток: `/` → (опц. `/clicker`) → `/match` → `/broadcast` (тоггл) → назад.

1. **`/` — Новый матч.** Тогглы формата (сеты, геймы, golden point, тай-брейк), редактируемые
   имена игроков команд A и B. Кнопка «Начать матч» создаёт матч в store и переходит на `/match`.
2. **`/clicker` — Кликер (макет).** Карточка подключённого устройства, выбор режима кнопок
   (2 кнопки раздельно / 1 кнопка по нажатиям), выбор владельца кликера, тестовая зона.
   Настройки сохраняются в `clickerStore`; **BLE — заглушка**.
3. **`/match` — Активный матч.** Живой счёт из состояния движка, SVG-диаграмма подачи/корта,
   кнопки «+ Очко A/B», «Отменить», статус кликера. При `status === 'completed'` — оверлей
   победителя с действиями (новый матч / трансляция).
4. **`/broadcast` — Трансляция.** Landscape, крупный счёт (`tabular-nums`), зеркалит то же
   состояние. Тап по экрану — выход назад на `/match`.

## 6. Дизайн-система (токены из макета)

- Фон: `#0a0b0a` (телефон), `#161718` (галерея/дев-превью), `#070807` (трансляция).
- Поверхности: `#101210`, `#121412`, `#0c0d0c`.
- Акцент (лайм): `#c6f24e`. Текст: `#f4f5f1` / `#eef0ea` / `#cdd1c9`.
- Приглушённый: `#8c918a`, `#71766f`, `#5c615a`, `#3a3f38`.
- Live-красный: `#ff5c38`.
- Шрифты: Archivo (дисплей/цифры), JetBrains Mono (моно-метки, аптайм, проценты).
- Анимации: `pulse` (мигание индикаторов), `ring` (пульс-кольцо у подающего/кликера) — портируются
  как Tailwind keyframes.
- Статус-бар и «dynamic island» рендерятся как часть мобильного макета.

## 8. Открытые допущения

- Порядок ротации подачи 4 игроков фиксируется при старте матча (A1 → B1 → A2 → B2) — стандартная
  трактовка для падела; при необходимости уточняется позже.
- Проект инициализируется как git-репозиторий (сейчас не репозиторий) для коммита спеки и работы.
