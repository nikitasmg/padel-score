# Голосовая озвучка трансляции (Web Speech)

**Дата:** 2026-06-29
**Статус:** дизайн утверждён, ожидает реализации

## Цель

Добавить голосовое объявление ключевых моментов матча на экране трансляции
(`/broadcast`): выигрыш гейма, выигрыш сета, смена сторон, победа в матче.
Первая версия — на Web Speech API (`window.speechSynthesis`), чтобы быстро
проверить, доступен ли русский голос на устройстве пользователя; в дальнейшем
это открывает путь к озвучке имён игроков.

## Решения и ограничения

- **Подход:** Web Speech API (а не заранее озвученные аудиофайлы). Плюс —
  динамический текст (имена игроков, любые фразы) без файлов. Минус — качество
  и наличие `ru-RU` голоса зависят от ОС устройства; iOS Safari требует
  «разблокировки» синтеза пользовательским жестом. Это осознанный выбор: цель
  первой версии — протестировать жизнеспособность русского TTS.
- **Service Worker не трогаем.** Голоса берутся из ОС, кэшировать нечего,
  офлайн-поведение определяется устройством.
- **Движок счёта не трогаем.** Точка подключения — существующая система
  событий `useScoreEvents` / `diffMatch`, на которой уже построены визуальные
  эффекты в `BroadcastEffects`.
- **Область первой версии:** только экран `/broadcast`.
- **По умолчанию озвучка выключена** — чтобы не включалась внезапно.

## Архитектура

Новый модуль `lib/voice/`, по образцу `lib/padel/`.

### `lib/voice/speak.ts` — обёртка над speechSynthesis

- `isSpeechSupported(): boolean` — наличие `window.speechSynthesis`.
- `getRussianVoice(): SpeechSynthesisVoice | null` — выбирает лучший `ru-RU`
  голос. Обрабатывает асинхронную загрузку: `getVoices()` на старте часто
  пуст, список приходит позже через событие `voiceschanged`. Выбранный голос
  кэшируется.
- `speak(text: string): void` — создаёт `SpeechSynthesisUtterance`, ставит
  русский голос, настроенные `rate`/`pitch`, добавляет в очередь синтеза.
- `warmUp(): void` — «разблокировка» синтеза: короткое (пустое/пробельное)
  произнесение в рамках пользовательского жеста. Нужно для iOS Safari, иначе
  последующие вызовы `speak()` молчат.

### `lib/voice/phrases.ts` — событие → текст

Чистая функция `announcement(event: ScoreEvent, match: MatchState): string | null`.

Приоритет (одно объявление за тик): матч > сет > гейм. Смена сторон —
отдельное событие, может прозвучать вместе с крупным (синтез ставит в очередь).

| Событие | Фраза |
|---|---|
| `matchWon` | `Матч! <имена команды> побеждают` |
| `setWonBy` | `Сет, <имена команды>` |
| `gameWonBy` | `Гейм, <имена команды>` |
| `endsChanged` | `Смена сторон` |

`<имена команды>` формируется из `teams[t].players[].name`
(например `«Иван и Пётр»`). Если имена пустые или дефолтные — фолбэк
`«команда N»` (N = индекс команды + 1). Все формулировки собраны в этом файле,
чтобы их легко править.

### `lib/voice/useVoiceAnnouncements.ts` — хук

- Вход: `match: MatchState`, `enabled: boolean`.
- Использует `useScoreEvents(match)` (`event`, `seq`).
- На крупном событии (гейм/сет/матч) и/или смене сторон формирует фразы через
  `phrases.ts` и вызывает `speak()`.
- **Страж по `seq`** (ref последнего озвученного `seq`), чтобы не дублировать
  объявления из-за повторных ре-рендеров и React StrictMode.
- При `enabled === false` ничего не произносит.

### Настройка вкл/выкл

Состояние озвучки хранится в `localStorage` (переживает перезагрузку),
по умолчанию выключено. Читается хуком, переключается кнопкой в UI.

### UI на `/broadcast`

- Маленькая кнопка-иконка 🔊/🔇 в углу (рядом с индикатором геймпада).
  - Первый тап по кнопке также вызывает `warmUp()` — тем же жестом
    разблокируем синтез на iOS.
- Рядом ссылка **«Проверить голос»** — произносит тестовую фразу
  (`«Проверка озвучки, гейм за командой один»`) и показывает статусом, нашёлся
  ли русский голос. Прямой способ проверить русский TTS на конкретном телефоне.
- Если `ru-RU` голос не найден — подпись «русский голос недоступен».
- Если `speechSynthesis` не поддерживается — кнопка скрыта/disabled.

## Крайние случаи

- Нет поддержки `speechSynthesis` → кнопка не показывается.
- Голоса грузятся асинхронно → ждём `voiceschanged`, кэшируем выбор.
- Заход на трансляцию посреди матча → `useScoreEvents` стартует с `prev=null`,
  лишних объявлений на старте нет.
- Дубли от ре-рендеров/StrictMode → страж по `seq`.

## Вне области первой версии

- Заранее озвученные аудиофайлы / офлайн-гарантия одинакового качества.
- Озвучка на экране `/match`.
- Озвучка имён игроков как отдельная настройка (структура фраз это уже
  поддерживает, но как опция — позже).
- Озвучка каждого розыгрыша и текущего счёта (40-30 и т.п.).
