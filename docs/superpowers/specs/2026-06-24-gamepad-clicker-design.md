# Геймпад как кликер + чистка макета

Дата: 2026-06-24

## Цель

Заменить выдуманное BLE-устройство «Padel Clicker P1» реальным источником
кликов — игровым геймпадом (DualShock 4), подключённым к iPhone, через
**Gamepad API**. Геймпад управляет счётом активного матча. Попутно — несколько
правок UI на `/match` и удаление бутафории «вид телефона» из макета.

## Контекст и ограничения

- Стек: Next.js 16 (App Router), React 19, Zustand (persist в localStorage).
- Движок матча: `lib/padel/engine.ts`, состояние в `store/matchStore.ts`
  (`point(team)`, `undoPoint()`, `clear()`).
- Сейчас `store/clickerStore.ts` содержит заглушки: `connected: true`,
  `battery: 82`. Страница `/clicker` — макет BLE-устройства.
- **Gamepad API не событийный**: состояние кнопок нужно опрашивать в цикле
  `requestAnimationFrame`. `rAF` останавливается, когда PWA уходит в фон или
  экран гаснет. → Сценарий: телефон лежит экраном вверх у корта, экран
  включён. Удерживаем экран через **Wake Lock API**.
- **Safari quirk**: геймпад не появляется в `navigator.getGamepads()` пока
  после фокуса страницы не нажата любая кнопка. До этого статус —
  «нажмите любую кнопку».
- **Батарея недоступна**: Gamepad API в WebKit не отдаёт уровень заряда.
  Индикатор батареи удаляем везде, заменяем на статус подключения.

## Раскладка кнопок (DualShock 4, standard mapping)

- **L1** (`buttons[4]`) → очко команде A (`point(0)`)
- **R1** (`buttons[5]`) → очко команде B (`point(1)`)
- **✕ / Cross** (`buttons[0]`) → отменить последнее (`undoPoint()`)

Режим — «2 кнопки раздельно» (существующий `buttonMode: "two"`). Режим «1 кнопка
по нажатиям» в рамках геймпада не реализуем; поле `buttonMode` в сторе не
удаляем, чтобы не ломать типы.

## Архитектура

Три слоя, каждый со своей ответственностью.

### `lib/gamepad/useGamepad.ts` — низкоуровневый хук

- Опрашивает `navigator.getGamepads()` в цикле `requestAnimationFrame`.
- Edge-detection: фиксирует **момент нажатия** (переход `pressed` false→true),
  а не зажатие. Хранит предыдущий снимок состояния кнопок.
- Слушает `gamepadconnected` / `gamepaddisconnected`.
- Возвращает: `connected: boolean`, `gamepadId: string | null`, и принимает
  колбэк `onButtonDown(index: number)`.
- Не знает ничего про падел.

### `lib/gamepad/mapping.ts` — чистая функция маппинга

- `buttonToAction(index: number): "pointA" | "pointB" | "undo" | null`
- Изолирована от рендера → покрывается юнит-тестом.

### `lib/gamepad/useWakeLock.ts` — удержание экрана

- `navigator.wakeLock.request("screen")`.
- Переполучение блокировки на `visibilitychange` (после возврата из фона).
- Graceful degradation, если API недоступен (молча ничего не делает).

### `components/GamepadController.tsx` — связующий слой

- Монтируется глобально в `app/layout.tsx` (рендерит `null`).
- Использует `useGamepad` + `useWakeLock`.
- На `onButtonDown` вызывает `buttonToAction`, и если матч активен
  (`match !== null` и `status === "in_progress"`):
  - `pointA` → `point(0)`, `pointB` → `point(1)`, `undo` → `undoPoint()`.
  - Пишет человекочитаемое событие в `clickerStore.setLastEvent`
    (напр. «L1 · +1 команде A»).
- Обновляет `clickerStore`: `connected`, `gamepadId` из `useGamepad`.
- Wake Lock активен, пока матч в процессе.

## Изменения в сторе

`store/clickerStore.ts`:

- Удалить `battery`.
- `connected: boolean` — теперь реально обновляется (старт `false`).
- Добавить `gamepadId: string | null` и сеттеры `setConnected`, `setGamepadId`.
- `buttonMode` оставить (= `"two"`).
- Сброс persist-ключа не требуется; недостающие поля доберутся дефолтами.

## Изменения в движке / matchStore

Добавить принудительное завершение матча.

`lib/padel/engine.ts`:

- `finishMatch(state): MatchState` — определяет победителя по текущему счёту:
  сравнение по `sets`, при равенстве — по `games[currentSet]`, при равенстве —
  по `points`. Ставит `status = "completed"` и `winner`. Если всё равно —
  `winner` остаётся `undefined` (ничья).

`store/matchStore.ts`:

- `finish: () => void` — применяет `finishMatch` к текущему матчу.

## Изменения UI

### Удаление «вида телефона»

- Удалить компоненты `components/PhoneScreen.tsx` и `components/StatusBar.tsx`.
- Страницы `/` и `/clicker` рендерят контент **на всю ширину** (без
  `max-w-[440px]`), как уже сделано на `/match` и `/broadcast`.
- Фоновый радиальный градиент (был в `PhoneScreen`) перенести на `body` в
  `app/globals.css`, чтобы страницы не стали плоско-чёрными.
- Убрать компенсацию высоты фейкового статус-бара (`calc(100vh - 36px)` →
  `100vh` / `min-h-screen`, выверить отступы `pt-*`).

### `/clicker`

- Карточка устройства: реальный `gamepadId` + статус (**Подключён** /
  **Нажмите любую кнопку**) вместо строки «2 кнопки · BLE» и батареи.
- Блок «Назначение кнопок» отражает фактическую раскладку: L1 → A, R1 → B,
  ✕ → отмена. (Блок выбора режима «1 кнопка» убрать или сделать неактивным —
  не поддерживается геймпадом.)
- Блок «проверка»: `lastEvent` наполняется реальным обработчиком из
  `GamepadController` — нажал L1, увидел «L1 · +1 команде A».

### `/match`

- `components/ScoreButtons.tsx`: убрать проп `battery`; вместо «Кликер · NN%»
  — индикатор подключения геймпада (Подключён / Нет геймпада), цвет по статусу.
- **Кнопка «Завершить матч»** (видимая, во время активного матча): по нажатию
  — подтверждение, затем `finish()` → открывается существующий
  `MatchCompleteOverlay` с текущим счётом и победителем по сетам.
- **Кнопка трансляции**: вместо «⋯» — заметная подписанная кнопка
  (иконка + «Трансляция»), ведёт на `/broadcast`.

### `/broadcast`

- В нижней полосе заменить «Кликер · NN%» на статус геймпада (или убрать,
  если мешает) — `battery` из стора удалён.

### `MatchCompleteOverlay`

- Обработать случай `winner === undefined` (ничья при ручном завершении):
  показать «Ничья» вместо имени команды, не возвращать `null` если
  `status === "completed"`.

## Тестирование

- `lib/gamepad/__tests__/mapping.test.ts` — `buttonToAction` для индексов
  4/5/0 и прочих (→ null).
- Edge-detection в `useGamepad` — юнит-тест на мок-объекте gamepad: нажатие
  фиксируется один раз, удержание не триггерит повтор.
- `finishMatch` — юнит-тест на определение победителя (по сетам / геймам /
  очкам / ничья).
- Wake Lock и реальный опрос `rAF` — браузерные API, юнитами не тестируем;
  ручная проверка на iPhone с геймпадом PS4.

## Вне рамок (YAGNI)

- Режим «1 кнопка по нажатиям» для геймпада.
- Работа кликера в фоне / при погашенном экране (требует нативного приложения).
- Уровень заряда геймпада (нет в API).
- Настраиваемый ремаппинг кнопок (фиксированная раскладка).
